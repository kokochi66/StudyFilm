4. JSP 고급사용

더 복잡한 JSP 웹사이트 구성의 내용

1단계고급사용1

  • 쿠키
  • 자바빈
    1
  • 쿠키는 클라이언트에 데이터를 저장하는 형태의 객체로, 별도의 생성이 필요합니다.
    서버와 클라이언트간 요청과 응답 후에 관계가 끊어져도 서로의 정보를 유지할 수 있는 수단이 됩니다.
    실습을 위해서 두개의 jsp파일을 생성합니다.
  • 먼저 첫번째 파일은 위와같이 쿠키를 생성한 뒤, response객체에 담아줍니다.
    응답객체에 쿠키가 포함되어, 클라이언트에 저장되며, 시간은 60초 * 60으로 1시간이 됩니다.
  • 두번째 파일은 저장한 쿠키를 확인하는 파일로, 쿠키는 지정한 쿠키이외에 기본 쿠키가 존재하므로,
    모든 쿠키를 불러온 후, 지정한 쿠키의 id값으로 원하는 쿠키를 찾는 작업이 필요합니다.
    쿠키를 찾으면 해당 쿠키의 Value값을 가져와서 출력 후에, 저장시간을 0으로 지정하여 삭제시켜줍니다.
  • 첫번째 파일을 시작하면, 위와같은 링크가 존재합니다.
    현재 화면에서는 쿠키가 생성되었습니다.
  • 링크를 클릭하여 이동하면, 지정한 쿠키의 value값이 정상적으로 출력되는것을 확인할 수 있습니다.

    1
  • 자바 빈은 자바의 클래스 객체를 JSP파일 내부에서 사용할 수 있도록 하는 액션태그의 일종입니다.
    해당 기능을 사용하기 위해 먼저, 자바 클래스 객체를 하나 생성해줍니다.
  • alt + shift + s 버튼을 눌러서 위와같은 창을 열고, getter와 setter 추가 창을 엽니다.
  • 객체의 모든 값의 getter와 setter메소드를 추가해줍니다.
  • 각 getter와 setter들이 추가된것을 확인할 수 있습니다.
  • 다음으로, 실습을 위해 JSP파일을 생성하여 위와같은 액션태그를 작성합니다.
    id값은 자유롭게 설정하며, class는 생성한 클래스의 경로를 작성합니다.
    마지막으로 scope는 해당 자바빈의 사용 범위로, page, request, session, application 등 옵션이 존재합니다.
  • 그리고 나서 위와같이 해당 자바빈의 값을 setProperty로 설정해주고,
    getProperty를 사용하여 해당 적용한 값을 불러와 출력하도록 합니다.
  • 해당 JSP 파일을 실행하면 정상적으로 적용값이 출력되는 것을 확인할 수 있습니다.

2단계표현식

  • EL태그
  • 내장객체
    1
  • JSP에서는 내부에서 사용할 수 있는 기본 객체가 있다.
    첫번째 객체로 request객체는 웹브라우저에서 서버로 요청된 데이터를 관리하는 객체이다.
    실습을 위해서 위와같이 두개의 JSP파일을 생성한다.
  • 첫번째 파일에서는 두번째 파일로 값을 전달하기 위한 form태그를 작성한다.
  • 두번째 파일에서는 전달받은 값을 받아오며, 또한 request기본 메소드를 확인하기 위한 다양한 메소드를 출력한다.
  • 첫번째 파일을 실행하여 form태그에 값을 입력해넣는다.
  • 실행하면, 정상적으로 입력값이 전달되며, 기본 메소드를 통해서 어떤값이 출력되는지를 확인할 수 있다.

    1
  • 두번째로 response객체는 웹 브라우저로부터 받은 요청을 응답하는 역할을 하는 객체이다.
    실습을 위해, 또 하나의 JSP파일을 생성한다. 이번 실습은 이전 request를 전달하는 form에서 15세 미만은 이용할 수 없도록 만드는 실습이다.
    출력문을 위와같이 작성한다.
  • 두번째 페이지로 돌아가서, 위와같이 age값이 15미만이라면 새로 생성한 세번째 페이지로 이동하도록 메소드를 만든다.
  • 첫번째 파일로 돌아가 실행하여 나이를 15세 미만으로 작성한다.
  • 정상적으로 두번째 파일이 아닌, 세번째 파일로 이동하는것을 확인할 수 있다.
