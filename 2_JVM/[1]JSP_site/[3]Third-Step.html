1단계JSP태그

  • 스크립트릿
  • 선언
  • 표현식
  • 지시자
  • 주석
    1
  • 스크립트릿은 JSP 내부에서 자바코드를 작성할 때 사용하는 태그이다.
    위와같은 형태로 작성하여 자바코드를 작성할 수 있다.
  • 위와같은 형태로, 변수를 선언하거나, 반복문, 조건문 등을 사용할 수 있다.
  • 1
  • 2
    1
  • 선언문태그를 이용하면 JSP파일 내부에서 변수나 메소드르 선언할 수 있다.
    선언 태그를 이용하여 선언된 변수나 메소드는 전역의 의미로 사용된다.
  • 1
    1
  • 표현식을 이용하면 정의된 메소드나 변수를 print없이 바로 출력시킬 수 있다.
    위와같이 선언 시, 적용되어있는 메소드의 출력값이 나오게된다.
  • JSP파일을 실행하면 작성한 자바코드의 결과값이 정상적으로 출력되는 것을 볼 수 있다.
  • 1
  • 2
    1
  • 지시자 태그는 JSP파일 내부에서 전체적인 속성을 결정할 때 사용되는 태그이다.
    첫번째로 page로 사용될 때는 위와같이 언어설정이나 import설정을 할 때 사용된다.
  • 두번째로 include로 사용되는 경우에, 다른 문서의 내용을 가져와서 출력할 수 있다.
  • 기존의 jsp파일 내용이 출력되고, 사이에 별도의 include로 추가된 부분이 출력된다.
    추가로, taglib지시자는 uri, prefix속성을 사용하여 태그 라이브러리를 사용할 수 있게 해준다.
  • 1
  • 2
  • 3
    1
  • JSP에서 주석은 html의 주석과 별도로 사용되며, JSP의 주석은 실행 시,
    컴파일되지 않아 출력되는 html의 소스에서 표시되지 않는다.
  • 실행 후, 코드소스를 확인하면 JSP의 주석은 표시되지 않는 것을 확인할 수 있다.

4단계내부객체2

  • 이클립스 설정
  • JSP 시작하기
    1
  • 이클립스에서 Server탭에서 새로운 서버를 추가합니다.
  • 서버는 톰캣 7 버전으로 선택하고, Next를 눌러줍니다.
  • 톰캣 폴더 설정에서 다운로드 받은 톰캣의 경로를 넣어주고, 서버를 생성합니다.
  • 생성한 서버를 더블클릭하여 들어간 뒤,
    Server location = Use Tomcat installation 선택 , 서버 옵션에서 Publish module contexts to separate XML files 체크,
    마지막으로 HTTP 포트번호를 기존 8080에서 겹치는것을 피하기 위해서 8181로 지정하고 저장합니다.
  • 서버 시작버튼을 눌러서 서버를 실행시킵니다.
  • 웹 브라우저에서 localhost:8181을 입력하여, 서버가 정상 실행되는지 확인합니다.
