  • 테스트를 위해 doGet에 Hello world를 출력하도록 하고 ctrl + f11 로 실행합니다.
  • 콘솔에 Hello world가 출력된 것을 확인합니다.

  • 실행된 url이 web.xml에서 지정한 값으로 바뀐것을 확인합니다.

두번째데이터 전달

  • doGet
  • doPost
    1
  • HTML에서 Servlet으로 데이터를 전달하는데에는 form태그가 사용됩니다.
    doGet메소드는 method="get" 으로 전달된 경우에 실행되며, url을 통해 접근하는 메소드입니다.
    url이 그대로 노출되는 방식이므로 보안에 약점을 보이는 방식입니다.
  • 먼저 doGet메소드가 실행되면 출력되는 문구를 작성합니다.
  • 그 뒤, jsp에서 위와같이 form태그를 작성합니다.
    button을 클릭 할 시, 어노테이션 url로 get 메소드로 접근하는 역할을 하는 태그입니다.
  • ctrl + f11로 실행 후, 활성화된 버튼을 클릭합니다.
  • 클릭시, url로 접근이 되어 doGet 출력 문구가 정상적으로 출력되어있는지 확인합니다.
  • 또한, url을 그대로 직접 접근하더라도 doGet이 마찬가지로 출력되는지 확인합니다.
  • 1
  • 2
  • 3
  • 4
  • 5
  • 6
    1
  • doPost메소드는 form태그에서 method="post"로 전달된 경우에 실행됩니다.
    url로 정보가 나타나지 않고, header를 통해 전달되기 때문에 보안에 강점이 있습니다.
  • 먼저 doPost메소드가 실행되면 출력할 문구를 작성합니다.
  • jsp파일에서 위와같이 form태그를 method="post"로 작성합니다.
    버튼을 클릭하면 doPost메소드가 실행되도록 하는 역할을 하는 태그입니다.
  • ctrl + f11로 실행하여 나타나는 post 버튼을 클릭합니다.
  • 클릭시 url로 이동되며, doPost메소드가 정상적으로 실행되는지 확인합니다.
  • 여기서, url로 직접접근하면 doGet메소드가 정상적으로 실행되는지 확인합니다.

네번째파라미터

  • form 태그
  • 한글처리
    1
  • 이클립스에서 Server탭에서 새로운 서버를 추가합니다.
  • 서버는 톰캣 7 버전으로 선택하고, Next를 눌러줍니다.
  • 톰캣 폴더 설정에서 다운로드 받은 톰캣의 경로를 넣어주고, 서버를 생성합니다.
  • 생성한 서버를 더블클릭하여 들어간 뒤,
    Server location = Use Tomcat installation 선택 , 서버 옵션에서 Publish module contexts to separate XML files 체크,
    마지막으로 HTTP 포트번호를 기존 8080에서 겹치는것을 피하기 위해서 8181로 지정하고 저장합니다.
  • 서버 시작버튼을 눌러서 서버를 실행시킵니다.
  • 웹 브라우저에서 localhost:8181을 입력하여, 서버가 정상 실행되는지 확인합니다.

  • 2
  • 3
  • 4
  • 5
  • 6
    1
  • 이클립스의 Project Explorer에서 위와같이 Dynamic Web project를 생성합니다.
  • 프로젝트의 이름을 설정하고 Next로 이동합니다.
  • web.xml 생성에 체크하고 finish를 눌러 프로젝트를 생성합니다.
  • WebContent 폴더 안에 새로운 jsp파일을 생성합니다. 이름은 자유롭게 합니다.
  • body 태그 내부에 Hello world를 작성합니다.
  • ctrl + f11 단축키를 사용하여 서버를 작동시킵니다.
  • 브라우저에서 정상적으로 Hello world가 출력되는지 확인합니다.
