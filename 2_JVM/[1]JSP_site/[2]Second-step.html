두번째데이터 전달

  • doGet
  • doPost
    1
  • HTML에서 Servlet으로 데이터를 전달하는데에는 form태그가 사용됩니다.
    doGet메소드는 method="get" 으로 전달된 경우에 실행되며, url을 통해 접근하는 메소드입니다.
    url이 그대로 노출되는 방식이므로 보안에 약점을 보이는 방식입니다.
  • 먼저 doGet메소드가 실행되면 출력되는 문구를 작성합니다.
  • 그 뒤, jsp에서 위와같이 form태그를 작성합니다.
    button을 클릭 할 시, 어노테이션 url로 get 메소드로 접근하는 역할을 하는 태그입니다.
  • ctrl + f11로 실행 후, 활성화된 버튼을 클릭합니다.
  • 클릭시, url로 접근이 되어 doGet 출력 문구가 정상적으로 출력되어있는지 확인합니다.
  • 또한, url을 그대로 직접 접근하더라도 doGet이 마찬가지로 출력되는지 확인합니다.

  • doPost메소드는 form태그에서 method="post"로 전달된 경우에 실행됩니다.
    url로 정보가 나타나지 않고, header를 통해 전달되기 때문에 보안에 강점이 있습니다.

    버튼을 클릭하면 doPost메소드가 실행되도록 하는 역할을 하는 태그입니다.
  • ctrl + f11로 실행하여 나타나는 post 버튼을 클릭합니다.
  • 클릭시 url로 이동되며, doPost메소드가 정상적으로 실행되는지 확인합니다.
  • 여기서, url로 직접접근하면 doGet메소드가 정상적으로 실행되는지 확인합니다.

세번째생명주기

  • Servlet의 생명주기
  • 선처리와 후처리
    1
  • Servlet객체는 한번 생성되면, 사라질 때 까지 기존의 객체값을 유지합니다.
    doGet, doPost가 실행되면 기존 객체 내부에서 요청시마다 실행되어 속도에 이점이 있습니다.
  • 실습을 위해 새로운 Servlet 파일을 생성합니다.
  • 기존 doGet, doPost와 init(), destroy() 메소드도 생성해줍니다.
  • 각 메소드가 실행되면 출력될 출력 문구를 메소드마다 적어줍니다.
  • 서버를 실행하고, init()메소드가 가장 먼저 실행되는지를 확인합니다.
  • 새로고침을 누르면 doGet 메소드가 정상적으로 실행되는지 확인합니다.
  • Server탭에서 stop을 눌러, 서버를 멈춥니다.
  • 콘솔에서 destroy()메소드가 정상적으로 실행되었는지 확인합니다.

  • init()메소드 이전에 실행되는 선처리 메소드와 destroy()메소드 이후에 실행되는
    후처리 메소드를 각각 @PostConstruct, @PreDestroy로 정의할 수 있습니다.
  • java파일에서 위와같은 메소드를 생성하며, 어노테이션을 각각 지정해줍니다.
  • 서버 실행 시, init() 메소드 이전에 선처리 메소드가 실행되었는지 확인합니다.
  • 서버를 멈춘 후, destroy() 메소드 이후에 후처리 메소드가 실행되었는지 확인합니다.

네번째파라미터

  • form 태그
  • 한글처리
    1
  • JSP파일을 새로 생성하여, 위와같은 form태그를 작성합니다.
    form 태그의 action은 새로 만들 서블릿 파일의 어노테이션과 맞춰줍니다.
  • 새로운 서블릿을 생성하여, 위와같은 코드를 작성합니다.
    위의 코드는 form태그로 전달된 데이터를 request 객체로 받아오는 코드입니다.
    해당 내용을 PrintWriter를 이용하여 html태그로 출력합니다.
  • JSP파일을 실행하여, 입력값을 넣고 전송합니다.
  • 출력된 값이 잘 나오는지 확인합니다.
    request로 받아온 한글값은 한글설정을 해주지 않았기 때문에 깨져서 나오게 됩니다.

    1
  • doPost에서 읽어오는 한글을 제대로 받아오기 위해서 위와같은 설정을 해줍니다.
  • 설정 후에 다시 jsp파일로 돌아가서 똑같이 값을 제출합니다.
  • 이번에는 입력된 한글이 정상적으로 출력되는 것을 볼 수 있습니다.
  • 동일한 코드를 doGet에서도 실행하기 위해서, 콘솔 출력문과 함께, doGet으로 옮깁니다.
  • get메소드로 실행하기 위해서 form태그의 method를 get으로 바꾸어줍니다.
  • doGet에서는 request.setCharacterSet설정을 변경하여도 읽어들이는 한글이 깨져서 나오게됩니다.
  • 이를 해결하기 위해서, server.xml파일을 편집해야합니다.
  • server.xml파일에서 Connector에서 포트가 생성된 서버와 일치하는 태그를 찾은 후,
    URIEncoding="UTF-8"을 추가한 뒤, 실제 서버파일과 Syncronized시켜줍니다.
  • 설정 후, JSP파일을 실행하여 확인하면 doGet에서도 한글이 정상적으로 출력되는것을 확인할 수 있습니다.

다섯번째기타내용

  • 초기화 파라미터
  • 공유 파라미터
  • 웹어플 감시
    1
  • 초기화 파라미터는 생성된 서블릿 파일에 초기에 갖고있는 파라미터를 설정하는 것입니다.
    실습을 위해 새로운 서블릿 파일을 생성하여줍니다.
  • web.xml을 열어서, 위와같이 서블릿 매핑과 함께 파라미터값을 설정해줍니다.
  • 초기화 파라미터값을 가져오는 함수는 서블릿 기본 내장함수이므로 바로 불러올 수 있습니다.
    위와같이 파라미터값을 불러온 후, 콘솔에 값을 출력합니다.
  • 실행하면, 콘솔에 정상적으로 파라미터값이 출력되는것을 확인할 수 있습니다.
  • 다음은 어노테이션으로 초기화 파라미터를 설정합니다. 설정을 위해 web.xml의 설정값을 주석처리합니다.
  • 어노테이션에서 위와같이 각 파라미터의 name과 value를 설정해줍니다.
  • 출력하면 새롭게 바뀐 값으로 파라미터가 바뀌었으며, 어노테이션 경로로 바뀐것을 확인할 수 있습니다.
